## 개념

1. 문제 해결 방식 정의 (분석 접근법)
데이터를 처음 접했을 때, 분석의 목적에 따라 학습 유형을 결정해야 합니다.

* 지도 학습 (Supervised Learning): 입력 데이터와 함께 '정답(Label)'이 주어지는 경우입니다.

    * 분류(Classification): 범주형 답을 예측할 때 사용합니다 (예: 타이타닉 생존 여부 0 또는 1).

    * 회귀(Regression): 연속적인 수치를 예측할 때 사용합니다.

    * 이유: 새로운 데이터가 들어왔을 때 합리적인 출력값을 예측하기 위한 모델을 만들기 위함입니다.

* 비지도 학습 (Unsupervised Learning): 답(Label)이 없는 데이터에서 패턴이나 구조를 찾을 때 사용합니다 (예: 군집화).

    * 이유: 데이터 내에 숨겨진 유사한 그룹을 찾아내거나 데이터의 특성을 파악하기 위함입니다.


2. 데이터 전처리 (Data Preprocessing)데이터를 분석 가능한 형태로 만드는 과정으로, 분석 결과의 신뢰도를 결정짓는 가장 중요한 단계입니다.

* 결측치(Missing Value) 및 데이터 정보 확인: info() 함수를 사용하여 각 컬럼의 데이터 타입과 결측치 존재 여부를 파악합니다.

    * 이유: 결측치는 분석 모델에 오류를 일으키거나 결과를 왜곡할 수 있으므로, 삭제하거나 적절한 값으로 대체해야 합니다.

* 문자형 변수의 수치화 및 그룹화: map()이나 replace()를 사용하여 텍스트 데이터를 숫자나 명목 변수로 변환합니다.

    * 예시: 타이타닉 데이터에서 승객의 이름(name)에서 호칭(title)을 추출하여 'Special', 'Mr', 'Mrs' 등으로 그룹화한 뒤 수치로 변환합니다.

    * 이유: 대부분의 기계학습 알고리즘은 숫자 데이터만 처리할 수 있으며, 복잡한 문자열에서 핵심 정보(예: 사회적 지위를 나타내는 호칭)만 뽑아내어 분석의 효율성을 높이기 위함입니다.

* 문자열 정제: strip()으로 공백을 제거하고, split()으로 필요한 부분만 분리합니다.

    * 이유: 데이터 수집 과정에서 발생한 불필요한 공백이나 문장 부호를 제거하여 데이터의 일관성을 확보하기 위함입니다.


3. 데이터 구조 조작 및 변수 활용데이터의 형태를 바꾸어 다각도로 분석하는 단계입니다.

* 데이터 합치기 (concat): 여러 개의 데이터프레임을 행(위아래) 또는 열(옆)로 합칩니다.

    * 이유: 훈련 데이터(train)와 테스트 데이터(test)를 구분하거나, 분산된 정보를 하나로 모아 전체적인 분석을 수행하기 위함입니다.

* 인덱스 재설정 (set_index, reset_index): 특정 컬럼을 인덱스로 만들거나, 기존 인덱스를 일반 컬럼으로 되돌립니다.

    * 이유: 데이터를 그룹화하거나 정렬한 후, 인덱스를 초기화하여 데이터 구조를 단순하게 만들고 다음 분석 단계로 넘어가기 위함입니다.

* 데이터 재구조화 (stack, unstack): 열을 행으로(stack) 또는 행을 열로(unstack) 변환합니다.

    * 이유: 계층적 인덱스(Multi-index) 구조에서 특정 변수 간의 관계(예: 성별에 따른 생존율 변화)를 한눈에 비교하기 쉬운 표 형태로 만들기 위함입니다.


4. 분석 흐름 및 결과 활용데이터를 가공한 후 어떤 인사이트를 도출할지 설계합니다.

* 그룹별 통계 분석: groupby()를 활용해 특정 변수별 평균이나 빈도를 계산합니다.

    * 논리적 전개: "성별(sex)과 생존 여부(survived)에 따라 데이터의 평균값을 확인하여, 어떤 집단이 더 높은 생존 확률을 가졌는지 분석한다".

* 변수 선택 (Feature Selection): 분석에 불필요한 변수(예: 유니크한 값이 너무 많은 name 열)는 삭제하고, 핵심적인 정보를 담은 변수(title, age, sex 등)만 선택합니다.

    * 이유: 모델의 복잡도를 줄이고 예측 성능을 높이기 위함입니다.

시험 대비 팁: 서술형 답안 작성 시 "이 데이터는 결측치가 많으므로 info()로 확인한 뒤, map()을 사용하여 중요한 특징만 추출하여 명목 변수화한다. 이는 모델이 문자를 이해하지 못하기 때문에 숫자로 변환하여 분석 효율을 높이기 위함이다"와 같이 [조치 - 방법 - 이유]의 흐름으로 서술하시기 바랍니다.


## 서술형

1. 문제 정의 및 분석 방향 설계 (Logic: 목적에 맞는 방법론 선택)
데이터의 'Target(정답)' 컬럼 유무와 데이터의 형태를 보고 분석 방향을 결정해야 합니다.

* 지도학습(Supervised Learning) 선택 이유:

    * 데이터에 명확한 정답(예: 생존 여부, 주가, 등급)이 포함되어 있다면, 과거 데이터를 통해 입력과 출력의 상관관계를 학습하여 새로운 데이터의 결과를 예측하기 위함입니다.

    * 분류(Classification): 정답이 'A/B', '합격/불합격'처럼 범주형일 때.

    * 회귀(Regression): 정답이 '가격', '온도'처럼 연속적인 수치일 때.

* 비지도학습(Unsupervised Learning) 선택 이유:

    * 정답(Label)이 없을 때 데이터 자체의 유사성이나 패턴을 찾아내기 위함입니다. 고객 세그먼트 분류(군집화) 등에 활용합니다.

2. 데이터 전처리 전략 (Logic: 데이터 신뢰성 확보)
전처리는 "모델이 학습하기 좋은 깨끗한 상태"를 만드는 과정입니다.

* 결측치(Missing Value) 처리 전략:

    * 삭제(dropna): 결측치가 전체 데이터에 비해 매우 적거나, 해당 행의 정보가 불완전하여 분석에 방해가 될 때 선택합니다.

    * 대체(fillna): 데이터 손실을 막기 위해 평균값, 중앙값 혹은 도메인 지식에 근거한 특정 값으로 채웁니다.

    * 논리: "데이터의 통계적 유의성을 유지하면서 표본 수를 최대한 확보하기 위해 중앙값으로 대체한다."

* 데이터 병합(concat):

    * 분석을 위해 훈련 데이터와 테스트 데이터를 동일한 기준으로 전처리해야 할 때, 혹은 여러 출처의 데이터를 하나로 묶어 종합적인 상관관계를 파악해야 할 때 사용합니다.

* 인덱스 관리(set_index, reset_index):

    * 특정 컬럼(예: 날짜, ID)을 기준으로 데이터를 조회하거나 그룹화할 때 인덱스로 지정하고, 계산이 끝난 후 다시 일반 컬럼으로 되돌려 가시성을 높입니다.

3. 변수 선택 및 피처 엔지니어링 (Logic: 데이터의 가치 추출)
원시 데이터(Raw data)를 그대로 쓰지 않고 분석에 유리하게 가공하는 단계입니다.

* 문자형 변수의 수치화(map, replace, lambda):

    * 기계학습 알고리즘은 수학적 연산을 기반으로 하므로 텍스트(예: 성별 'male', 'female')를 숫자(0, 1)로 변환해야 합니다.

    * 논리: "범주형 데이터를 연산 가능한 수치형 데이터로 변환하여 모델이 패턴을 인식할 수 있게 한다."

* 복잡한 데이터의 그룹화(Nominalization):

    * 예를 들어, 타이타닉의 'Name' 컬럼에서 'Mr.', 'Mrs.', 'Master' 등의 호칭을 추출하는 경우입니다.

    * 논리: "단순한 이름 정보는 노이즈(Noise)가 될 수 있으므로, 사회적 지위나 성별을 함축하는 '호칭' 변수를 추출하여 분석의 변별력을 높인다."

* 불필요한 변수 제거:

    * 데이터의 고유값이 너무 많거나(예: ID 번호), 분석 목적과 상관없는 컬럼은 삭제하여 모델의 복잡도를 낮추고 과적합(Overfitting)을 방지합니다.

4. 데이터 재구조화 및 활용 (Logic: 인사이트 도출)
분석 결과를 시각화하거나 보고서 형태로 정리하기 위한 단계입니다.

* stack과 unstack의 활용:

    * Stack: 컬럼들을 행으로 내려서 데이터를 수직적으로 길게 만듭니다 (데이터 통합 분석 시 유리).

    * Unstack: 행 인덱스를 컬럼으로 올려서 표 형태(Wide format)로 만듭니다 (그룹 간 비교 시 유리).

    * 논리: "다중 인덱스 구조에서 특정 변수 간의 교차 분석(Cross-tabulation)을 수행하여 가독성을 높이고 인사이트를 쉽게 도출하기 위함이다."

* 분석 결과의 활용 방안:

    * 예: "도출된 생존율 데이터(결과)를 바탕으로, 특정 등급 및 성별의 생존 가능성이 높음을 확인하여 향후 사고 예방 정책의 우선순위 설정에 활용한다."


💡 서술형 답안 작성 예시 (팁)
질문: 주어진 데이터의 '성별'과 '이름' 컬럼을 어떻게 처리할 것인지 서술하시오.

답안 예시:

1. 현상 파악: 성별은 문자형 데이터이며, 이름은 고유한 문자열이 많아 그대로 학습하기 어렵다.

2. 처리 방법: 이름 컬럼에서 map과 lambda 함수를 활용해 'Mr', 'Miss'와 같은 호칭(Title)을 추출하고, 이를 4~5개의 그룹으로 범주화한다. 이후 성별과 추출된 호칭을 replace나 숫자로 매핑하여 수치형 변수로 변환한다.

3. 근거(이유): 모델은 숫자 데이터만 처리할 수 있기 때문에 수치화가 필수적이며, 이름 자체보다는 호칭에 담긴 사회적 지위나 성별 정보가 생존 여부(Target)를 예측하는 데 더 유의미한 변수(Feature)로 작용하기 때문이다. 또한, 불필요한 이름 정보는 삭제하여 모델의 복잡도를 줄인다.
